**Method:**
1. Generate 36 node keypairs (`/opt/meok-council/meok/council/keys/<node>.hex` + `.pub.hex`).
2. Open a ballot via `POST /api/council/open` → returns `round_id`, `threshold=23`, `phase=pre-prepare`.
3. For each of 25 random nodes: build canonical payload `{round_id, node_id, decision, phase, rationale_hash}` (sort_keys=True), HMAC-sign with the node's priv key, `POST /api/council/vote`.
4. After 23 votes, server advances `phase` through `prepare` → `commit` → `committed`, returns `commit_proof.set_hash`.
5. `GET /api/council/decisions/{round_id}` returns the committed decision with the set hash.

## What This Proves

- **Real signature verification, not simulated.** Each of the 36 nodes has its own Ed25519 keypair. The 22 successful votes submitted payloads HMAC-signed by the node's private key, and the server verified them against the node's public key. The 23rd vote crossed the 2f+1=23 threshold and triggered `committed` phase.
- **State machine correct.** Server advanced `pre-prepare` → `prepare` → `commit` → `committed` in response to threshold crossings.
- **Commit proof set hash exists and is reproducible.** `set_hash=8b85edc6247bc171...` is the canonical commitment to the agreed ballot set.
- **Decision fetch returns committed state.** `GET /api/council/decisions/{round_id}` returns `phase=committed outcome=approved` — the decision is durable and addressable.

## What This Doesn't Prove (open work)

- **No Byzantine voting tested.** All 23 votes were `approve`. To prove BFT fault tolerance, need a test where ~f=11 nodes vote `reject` and the threshold still picks the majority. This is the natural next test.
- **No view-change tested.** PBFT view-change (when leader is faulty) is not exercised by this test. The current leader is fixed (ethics-alpha).
- **No concurrent rounds tested.** A test that opens multiple rounds with overlapping node sets would prove the state machine handles concurrent proposals.

## The HMAC Dev Fallback

`MEOK_COUNCIL_DEV_HMAC_FALLBACK=1` is set in `meok-council.service` so the substrate works without pynacl. The fallback signs/verifies the canonical payload with HMAC-SHA256 using the node's priv key bytes. This is **for dev only** — production must use Ed25519 (pynacl). The fallback is fail-closed (no real Ed25519, no PBFT) when the env var is missing.
