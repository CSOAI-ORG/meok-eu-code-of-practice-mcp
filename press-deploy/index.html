How the keystone works

The MEOK keystone (meok-attestation-api.vercel.app) issues Ed25519-signed JSON-LD attestations. Each attestation carries a unique cert_id, the entity name, the framework reference (e.g. "Regulation (EU) 2024/1689 Article 50"), the issued_at + expires_at timestamps, and an Ed25519 signature. Anyone with the cert_id can verify the signature offline with pip install meok-attestation-verify.

Each attestation is also anchored to a public transparency log + hash-chained to a tamper-evident audit trail. The format survives 7-year retention requirements without rotting.
