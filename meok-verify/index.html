How MEOK signing works

Cert issued. MEOK's signing API (server-side HMAC-SHA256 key) signs a canonical JSON payload that includes regulation, entity, score, findings, dates and tier.
Public verify_url returned. The issuer gets a URL like meok-attestation-api.vercel.app/verify/MEOK-DORA-ABC that anyone can open.
Anyone verifies. The holder shares the cert or the verify URL. The recipient paste-verifies here, calls the API directly, or runs pip install meok-attestation-verify offline.
Tampering breaks the signature. Any single-character change in the payload causes signature mismatch. A valid signature proves the payload wasn't altered.
365-day expiry. Every cert carries an expires_utc. Re-run the audit before that date to maintain continuous evidence.

What if MEOK disappears?
