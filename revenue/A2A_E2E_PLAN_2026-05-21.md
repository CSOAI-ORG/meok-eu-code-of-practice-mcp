# A2A E2E Substrate + Data Moat Strategy — 2026-05-21

## The thesis

The 12 agent-to-agent (A2A) MCPs we have in the Anthropic Registry are by far the highest income-feasibility cluster of the 47. Reasons:

1. **A2A is the next protocol layer the field is converging on.** Anthropic shipped MCP. Google shipped A2A. Stripe shipped A2A Payments. The pieces between agents need standardisation, and standardisation = rent.
2. **Per-call economics work.** Compliance MCPs sell as monthly subscriptions. A2A MCPs sell **per agent interaction** — billions of calls × tiny margin = real revenue.
3. **Geography-independent.** EU AI Act / DORA / NIS2 are EU-specific. A2A primitives are universal — identity, trust, audit, policy, rate-limit, handoff.
4. **Regulator-agnostic.** Whatever rules emerge, you still need to know which agent called which agent and what data was exchanged. The substrate is buy-side neutral.
5. **Data moat compounds.** Each call generates signed evidence. Aggregated, anonymised, it becomes proprietary threat intelligence + best-practice corpus.

## The E2E pipeline (sell this, not each MCP separately)

**One agent call traverses all 7 primary stages, generating 7 signed attestations:**

```
   Agent A                                                     Agent B
     │                                                            │
     │  1. identity-trust ─── verifies A's DID + trust score      │
     │  2. data-residency ─── checks transfer-basis (GDPR Ch.V)   │
     │  3. policy-enforcement ─── evaluate_call(A→B, scope)       │
     │  4. injection-firewall ─── scan prompt+RAG+args            │
     │  5. rate-limiter ─── grant token (sliding window)          │
     │  6. handoff-certified ─── sign provenance chain            │
     │  7. audit-logger ─── append hash-chained log entry         │
     │ ─────────────────────────────────────────────────────────► │
     │                                                            │
     │  ◄── response signed + chained back through audit-logger ──│
```

Each stage outputs a signed attestation. The 8th MCP (`a2a-governance-bridge`) folds them into a single auditor-defensible event for EU AI Act Art 12 + DORA Art 17 + ISO 42001 clause 9 evidence.

**Commercial positioning: "MEOK A2A Substrate — 7 primitives, 1 signed event, 1 invoice."**

## The 5 data moats (built passively as customers use the substrate)

Every customer using the A2A Substrate emits signed events. **We don't read the payloads** (that would be a privacy red flag) — we aggregate the **metadata** of what's happening across the fleet.

### Moat 1 — Prompt Injection Threat Intelligence (from `agent-prompt-injection-firewall-mcp`)

- Every blocked injection attempt → anonymized signature → aggregate corpus
- After 1M scanned prompts: we have the most current real-world prompt-injection dataset in industry
- **Sell as:** "MEOK Threat Intel Feed" — £999/mo to CISO teams + SIEM vendors (Wazuh, Elastic, Splunk integrations)
- **First milestone:** 50K scans → publish a public threat report (PR halo)

### Moat 2 — Policy-Violation Taxonomy (from `agent-policy-enforcement-mcp`)

- Every `evaluate_call → DENY` event → categorise the violation reason
- Build the canonical "agent permissions gone wrong" taxonomy
- **Sell as:** Quarterly "State of Agent Permissions" report (free for press; £2,499 for full data access)
- **First milestone:** 100K decisions logged → first quarterly report

### Moat 3 — Incident-Clock Graph (from `agent-audit-logger-mcp` + `ai-incident-reporting-mcp`)

- Every incident maps to multiple regulatory clocks (EU AI Act Art 73, DORA Art 19, NIS2 Art 23, GDPR Art 33)
- Aggregate cross-regime patterns: "how often does an Art 73 fire alongside Art 19 alongside Art 33?"
- **Sell as:** Consulting hours at £2,500/day to firms that need cross-regime mapping
- **First milestone:** 100 cross-regime incidents → publish a 1-page "regulatory event correlation matrix"

### Moat 4 — Best-Practice Handoff Patterns (from `agent-handoff-certified-mcp`)

- See how production agent fleets actually delegate work between Claude / GPT / Gemini / Llama
- Anonymized: "78% of high-trust handoffs go Claude→GPT, 22% reverse; care-sensitive tasks favour local Ollama"
- **Sell as:** Annual "State of A2A" report — content marketing engine that drives top-of-funnel
- **First milestone:** 10K handoffs → first quarterly trend report

### Moat 5 — Trust Network Graph (from `agent-identity-trust-mcp`)

- The set of agent DIDs that have ever talked to each other → trust graph
- Adversarial agent identification through anomalous interaction patterns
- **Sell as:** "MEOK Agent Reputation Score" — £0.0005/check, called by firewalls + payment systems before completing A2A transactions
- **First milestone:** 1K unique DIDs registered → public trust-graph stats page
