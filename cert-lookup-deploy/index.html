The full verification chain

For maximum security, you can verify the attestation offline:

pip install meok-attestation-verify
meok-attestation-verify < cert.json

What's in the JSON-LD payload

  • cert_id — the unique ID (MEOK-REGULA-XXXXX)
  • entity — what the attestation is for (e.g. "Monzo ML fraud detection")
  • framework — which framework (eu-ai-act, dora, nis2, etc)
  • regulation — the specific regulation referenced
  • issued_at — when it was signed (ISO 8601 UTC)
  • expires_at — when the attestation expires (1 year default)
  • signature — Ed25519 signature, 64 bytes, base64 encoded
  • public_key — the MEOK public key that signed it
  • hmac_chain — the HMAC chain anchor (for tamper-evidence)
