# TC260 Website Analysis for COAI Replication

## TC260 Organizational Structure

Based on the research, TC260 has a hierarchical structure:

### Main Body
- **TC 260** - Main committee
- **Secretariat** - Administrative coordination

### Working Groups (WGs)
1. **WG1**: Information Security Standard System & Coordination
2. **WG2**: Classified Information System Security
3. **WG3**: Encryption Standards
4. **WG4**: Authentication & Authorization
5. **WG5**: Information Security Management
6. **WG6**: Communication Security
7. **WG7**: Information Security Management
8. **SWG-BDS**: Big Data Security (Special Working Group)

**Note**: WGs in dark blue allow foreign members

## Multi-Level Protection Scheme

TC260 organizes standards into three categories:

### 1. Standards on General Requirements
- Baseline for Cybersecurity Classified Protection
- Part 1: Security General Requirements
- Part 2: Cloud Computing
- Part 3: Mobile Interconnection
- Part 4: Internet of Things
- Part 5: Industrial Control System

### 2. Standards on Design Requirements
- Technical Requirements of Security Design
- Part 1: General Security Design
- Part 2: Network Security
- Part 3: Cloud Computing
- Part 4: Mobile Internet
- Part 5: IoT
- Part 6: Industrial Control

### 3. Standards on Testing and Evaluation
- Evaluation Requirements for Classified Protection
- Part 1: Security General
- Part 2: Network Security
- Part 3: Cloud Computing
- Part 4: Mobile
- Part 5: IoT
- Part 6: Industrial Control

## Key Website Features to Replicate

1. **Hierarchical Organization** - Clear structure of working groups
2. **Standards Database** - Searchable list of all standards
3. **Public Comment Periods** - Open feedback on draft standards
4. **Contact Information** - Named contacts for each standard
5. **Timeline Display** - Comment periods with dates
6. **Multi-Level Classification** - Tiered approach to requirements

## COAI Equivalent Structure

### Working Groups (Mapped to 33-Agent Council)
1. **Guardian Agents (11)** - Safety, Security, Privacy
2. **Arbiter Agents (11)** - Fairness, Transparency, Accountability
3. **Scribe Agents (11)** - Documentation, Compliance, Reporting

### Standards Categories (Mapped to Frameworks)
1. **EU AI Act** - Mandatory Western standard
2. **NIST AI RMF** - Voluntary US framework
3. **TC260** - China reference standard

### Multi-Level Classification (Risk-Based)
1. **Minimal Risk** - No requirements
2. **Limited Risk** - Transparency obligations
3. **High Risk** - Full compliance required
4. **Unacceptable Risk** - Prohibited
